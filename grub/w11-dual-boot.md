# USB Hardware Boot Key for Dual-Boot (GRUB Probe Method)

Automatically boot straight into Windows 11 when a specific USB flash drive is inserted at startup. When the USB is missing, it skips straight into Fedora with a minimal timeout delay.

## How it Works
Instead of making the USB drive bootable, the internal GRUB bootloader probes all connected storage devices for a specific dummy tag file (`boot_windows.tag`) early in the boot sequence.

## Step-by-Step Configuration

### 1. Extract the Windows Menuentry String
Find the exact name of the Windows boot entry generated by os-prober:
```bash
sudo grep -i "menuentry 'Windows" /boot/grub2/grub.cfg

```

*Example Output:* `Windows Boot Manager (on /dev/nvme1n1p1)`

### 2. Create the Custom GRUB Hook

Create a new executable script ahead of the standard menu entries:

```bash
sudo nano /etc/grub.d/01_usb_timeout

```

Paste the following script structure (ensure the `default` string matches step 1 exactly):

```bash
#!/bin/sh
cat << 'EOF'
# Scan for the trigger file on any connected storage device
search --no-floppy --file --set=usb_present /boot_windows.tag

if [ -n "$usb_present" ]; then
    # Match the exact string from step 1
    set default="Windows Boot Manager (on /dev/nvme1n1p1)"
    set timeout=0
else
    # Default to Fedora (Index 0) with a fast timeout
    set default=0
    set timeout=1
fi
EOF

```

### 3. Permissions & Compile

Make the script executable and rebuild the master GRUB configuration file:

```bash
sudo chmod +x /etc/grub.d/01_usb_timeout
sudo grub2-mkconfig -o /boot/grub2/grub.cfg

```

### 4. Prepare the USB Key

Create an empty file named exactly `boot_windows.tag` in the root directory of the flash drive:

```bash
touch /run/media/ouzx/YOUR_USB_NAME/boot_windows.tag

```

## 🚨 Emergency Safety Nets

If something goes wrong with the GRUB compilation or the script syntax, use these hardware overrides to bypass the automated loop:

1. **Motherboard Boot Menu (F9):** Turn on the laptop and immediately spam **F9** to access the HP boot menu. This bypasses GRUB entirely and lets you boot straight into your internal Fedora or Windows UEFI target.
2. **GRUB Intercept (ESC / Shift):** Spam **Escape** (or hold **Shift**) immediately after the POST screen. This tells GRUB to halt any automated countdown timers and forces the standard menu to stay on screen.
